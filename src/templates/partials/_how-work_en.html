We use nunjucks as a template processor to speed up the HTML code writing process and then support that code in the easiest and fastest way possible. To ensure your project’s scalability, we use the component-based approach described in the BEM methodology.

Twig

We make use of the Sass preprocessor for the CSS code. It allows using functions unavailable in CSS. For instance, variables, nesting, mixins, inheritance, and other awesome stuff. Yet in the end, you always get the CSS code that is fast to run and easy to work with.

Usually, we adhere to jQuery, with vanilla being our second option. We also take advantage of the module-based approach to write components but can also take advantage of additional patterns in case of more complicated tasks.

Having a wide experience in animation implementation, we know all the ins and outs of browser rendering to achieve from 40 to 60 frames per seconds (depending on your requirements and expectations).

We use the gulp/webpack bundle to make the build process comfortable. Version control is done with git. Naturally, we’re used to all modern web design tools: Photoshop, Sketch, Zeplin, Avocode and more.
